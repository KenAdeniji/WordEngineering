Domain Name System (DNS)

A computer on the Internet has both an IP (Internet Protocol) address, and a fully qualified domain name. A client, resolver, may request connection to another computer using its host name. The DNS server providing the translation, is known as the name server. The three main levels that make-up the DNS hierarchy, are:
  • Root-level domain (represented by a single period)
  • Top-level domain (represented by various suffixes, for example, .com, .org)
  • Second-level domain (represented by the organization's name, for example, Microsoft)
DNS groups host computer names by domain (or subdomain) into zones. Prior to installing DNS, ensure that TCP/IP is running, and that the host has a static IP address.

DNS Manager

The DNS Manager is a graphical tool for managing DNS entries. When you start the DNS manager and there is no connected DNS server, it may prompt the user to connect to a DNS server.

NsLookup.exe

The NsLookup.exe is a console application that prints information about the host/domain name. If you do not specify a DNS server, the default DNS server is used.

DnsCmd.exe

A console application for updating the DNS servers.
  • dnscmd [<ServerName>] /enumzones (Will lists the zones that exist on the specified DNS server.)
  • dnscmd [<ServerName>] /info [<Setting>] (Will lists registry settings at the DNS server level.)
